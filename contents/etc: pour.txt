# 무엇이 '나' 인가?
동일한 한 사람이기 위해서는 function, input, output 이 동일해야 한다. 사람으로서 function 은 뇌와 온몸의 기억 세포 등이 될 것이고 input 은 오감 등의 감각 기관을 통해 받아들이는 정보, output 은 신경계로 전파되는 전기 신호이다.
'만약 뇌를 기계로 옮기면 같은 사람으로 봐야 하는가?' 라는 질문을 한다면, function 은 동일하지만, copy 한 시점으로부터 input 과 output 이 다르기 때문에 서로 다른 지성체라고 할 수 있다. 게다가 function 의 구조는 같지만 구현이 다르다는 차이점도 있다.(물리적으로 다른 물질로 구성됨)
그리고 function, input, output 을 통해 '나' 를 정의했다면, 내 몸의 일부가 사라지는 것도 function, input, output 에 변화가 생긴 것이기 때문에, 사라지기 전의 나와 사라진 후의 나는 다른 지성체라고 할 수 있다. 다른 지성체이기 때문에 같은 외부 환경에 다른 선택을 하는 상황이 존재할 수 있다.
만약 뇌의 일부가 사라진다고 해도 같은 맥락으로 이해할 수 있다.

# [Common Instructions]
malloc 이 NULL 을 반환하는 경우에 segfault 등이 발생하지 않도록 하는 것은 해야 한다지만, dup(), pthread_mutex_init() 등의 함수가 에러로 인해 오작동하는 경우도 굳이 고려해야 할까요?
개개인이 이에 대한 답이 다를 수 있을 것 같은데요. 다양한 의견 답글로 달아주시면 감사하겠습니다!

대부분 과제에 있는 Common Instructions 를 보면 segfault, bus error 등으로 인해 **예상치 못하게** 프로그램이 종료되지 않아야 한다는 문장이 있습니다.
때문에 대부분 카뎃분들이 malloc() 은 반환값이 NULL 일 경우를 체크하여 비정상적인 작동 및 종료를 방지하는데요.
사실 에러로 인해 정해진 값을 반환하는 함수는 malloc() 뿐만이 아니라 라이브러리 함수와 시스템 콜 함수를 통틀어 굉장히 많습니다.
하지만 이런 함수에 대해 에러 가능성이 있다는 것을 어렴풋이는 알더라도 굳이 더 깊이 알아보고 적용하는 분은 보지 못했고, 에러 가능성이 있는 함수가 에러인 상황을 생각도 해보지 않은 경우도 많았습니다.
그 함수가 에러를 발생하여 특정값을 반환할 수 있다고는 하지만, 일반적인 환경에서는 에러가 발생하지 않기 때문에 굳이 저런걸 다 염두하고 코드를 작성해야 하는가 하고 생각하시는 분도 많은 것 같습니다.

저는 이 쓰레드로 인해 ‘에러로 인해 특정 값을 반환하는 모든 함수에 대해 굳이 더 생각을 해봐야하는가?’ 라는 지금까지는 불확실하고 애매했던 물음에 답이 되었으면 좋겠습니다.
